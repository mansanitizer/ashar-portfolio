The Infinite Wikipedia

Rabbithole is more than just a search engine; it's a personalized knowledge discovery engine designed to cure curiosity. In an era where information is abundant but scattered, Rabbithole aims to synthesize complex topics into digestible, infinitely explorable learning paths.

The core concept was simple: What if Wikipedia was written specifically for you, right now, responding to exactly what you're curious about?

"Discovery is not about finding what you know you're looking for, but stumbling upon what you didn't know you needed."

How It Works

Under the hood, Rabbithole orchestrates a complex dance of AI agents. When a user enters a topic, the system doesn't just "search" for it. Instead:

  • Intent Analysis: An agent analyzes the user's query to understand the depth and context desired.
  • Content Synthesis: Multiple LLMs work in parallel to draft a comprehensive overview.
  • Dynamic Linking: The most interesting part‚Äîkeywords in the generated text become "portals" or "rabbit holes" to deeper content. Clicking one spawns a new generation task instantly.

Technical Challenges

State Management & Context

Keeping track of a user's journey through multiple levels of "rabbit holes" required a robust state management system. We utilized a graph-based data structure to track the relationship between topics.

Latency Optimization

Generating high-quality content takes time. To keep the experience fluid, we implemented aggressive streaming and optimistic UI updates. We use Vercel AI SDK for streaming responses to ensure the user feels immediate feedback.

// Simplified example of the streaming architecture
export async function POST(req: Request) {
  const { messages } = await req.json();
  const response = await openai.createChatCompletion({
    model: 'gpt-4',
    stream: true,
    messages
  });
  return OpenAIStream(response);
}

Mobile Adaptation

Bringing this experience to mobile was crucial for "on-the-go" learning. We used Capacitor to wrap our Next.js web application into a native Android app.

Key adaptations included:

  • Touch-optimized interactions (swipe to save, tap to explore).
  • Native sharing capabilities.
  • Offline caching for previously visited "holes".

Future Roadmap

We are currently exploring multi-modal learning paths, where the AI can generate not just text, but relevant images (using Flux or Midjourney APIs) and even short audio summaries.
